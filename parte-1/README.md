# Parte 1 - Figma para código  
## 🎯 Objetivo do Projeto
O desafio consistiu na conversão fiel de um design do Figma para código funcional, focando em precisão visual e qualidade técnica. O foco principal foi a reprodução das quatro seções iniciais da landing page Positivus:

- Hero Section: Layout dinâmico com foco em conversão.
- Services: Grade de serviços com elementos gráficos detalhados.
- Let's Make Things Happen: Seção de CTA (Call to Action) com design integrado.
- Case Studies: Estrutura de dados e exibição de cases.

## 🛠️ Tecnologias e Decisões Técnicas
Para este projeto, optei por uma stack moderna e performática:

- React + Vite: Para um ambiente de desenvolvimento ultra-rápido e bundling eficiente.
- CSS (Vanilla/Modules): Escolhi priorizar o uso de CSS puro para demonstrar domínio técnico em estilização, manipulação de layouts complexos.
- Responsividade: Implementação manual de breakpoints para assegurar que a interface seja indistinguível do design em qualquer tamanho de tela.

## 🚀 Diferenciais da Entrega
- Fidelidade Visual: Atenção rigorosa à tipografia, espaçamentos (paddings/margins) e paleta de cores original.
- Componentização: Estrutura de código organizada seguindo princípios de Clean Code, facilitando a manutenção e escalabilidade.
- Semântica HTML: Uso de tags semânticas para melhor acessibilidade e SEO.

## 🏁 Como Rodar o Projeto
```bash
// Instale as dependencias
npm install

// Gere o build de produção
npm run build

// Inicie o servidor local
npm run preview
```
